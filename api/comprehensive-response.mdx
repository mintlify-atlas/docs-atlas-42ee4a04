## Overview

The comprehensive response format is returned when `analysis_mode` is set to `"comprehensive"`. This mode provides in-depth analysis including key findings, timelines, multiple perspectives, quotes, and verification status in addition to raw search results.

## Response Structure

<ResponseField name="query" type="string" required>
  The original search query that was executed
</ResponseField>

<ResponseField name="analysis_mode" type="string" required>
  The analysis mode used for this search. Always `"comprehensive"` for comprehensive responses
</ResponseField>

<ResponseField name="comprehensive_analysis" type="string" required>
  Detailed analysis of the search topic with context and implications (typically 500+ words)
</ResponseField>

<ResponseField name="key_findings" type="array" required>
  Array of key finding objects representing the most important discoveries
  
  <Expandable title="Key Finding Object">
    <ResponseField name="category" type="string" required>
      The category of this finding. One of:
      - `"main_story"` - Primary development or main narrative
      - `"development"` - Secondary developments
      - `"context"` - Background information
      - `"impact"` - Consequences and effects
    </ResponseField>
    
    <ResponseField name="title" type="string" required>
      Concise title of the finding
    </ResponseField>
    
    <ResponseField name="content" type="string" required>
      Detailed explanation with specifics
    </ResponseField>
    
    <ResponseField name="sources" type="array" required>
      Array of source URLs supporting this finding
    </ResponseField>
    
    <ResponseField name="confidence" type="string" required>
      Confidence level of this finding. One of: `"high"`, `"medium"`, `"low"`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timeline" type="array" required>
  Chronological array of significant events related to the search topic
  
  <Expandable title="Timeline Event">
    <ResponseField name="date" type="string" required>
      Event date in `YYYY-MM-DD` format
    </ResponseField>
    
    <ResponseField name="event" type="string" required>
      Description of what occurred
    </ResponseField>
    
    <ResponseField name="source" type="string" required>
      The source reporting this event
    </ResponseField>
    
    <ResponseField name="significance" type="string" required>
      Explanation of why this event matters
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="direct_quotes" type="array" required>
  Array of relevant quotes from authoritative sources
  
  <Expandable title="Quote Object">
    <ResponseField name="quote" type="string" required>
      The exact quote from the source
    </ResponseField>
    
    <ResponseField name="speaker" type="string" required>
      Name of the person who said or wrote the quote
    </ResponseField>
    
    <ResponseField name="context" type="string" required>
      When and where the quote was made (e.g., "During a press conference on Monday")
    </ResponseField>
    
    <ResponseField name="source_url" type="string" required>
      URL of the source containing the quote
    </ResponseField>
    
    <ResponseField name="significance" type="string" required>
      Why this quote is important to the analysis
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="related_context" type="string" required>
  Historical background and contextual connections to help understand the topic
</ResponseField>

<ResponseField name="multiple_perspectives" type="array" required>
  Array of different viewpoints on the topic
  
  <Expandable title="Perspective Object">
    <ResponseField name="viewpoint" type="string" required>
      Label for this perspective (e.g., "Supporters", "Critics", "Industry Experts")
    </ResponseField>
    
    <ResponseField name="content" type="string" required>
      Analysis from this perspective
    </ResponseField>
    
    <ResponseField name="sources" type="array" required>
      Array of source URLs representing this perspective
    </ResponseField>
    
    <ResponseField name="reasoning" type="string" required>
      Explanation of the logic behind this viewpoint
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="implications" type="object" required>
  Analysis of consequences and impacts
  
  <Expandable title="Implications Object">
    <ResponseField name="short_term" type="string" required>
      Immediate consequences and near-term impacts
    </ResponseField>
    
    <ResponseField name="long_term" type="string" required>
      Potential long-term effects and lasting changes
    </ResponseField>
    
    <ResponseField name="stakeholders_affected" type="array" required>
      Array of groups, organizations, or entities impacted by this topic
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="verification_status" type="object" required>
  Analysis of information reliability and verification
  
  <Expandable title="Verification Status Object">
    <ResponseField name="confirmed_facts" type="array" required>
      Array of verified information with multiple reliable sources
    </ResponseField>
    
    <ResponseField name="unconfirmed_claims" type="array" required>
      Array of claims that lack sufficient verification
    </ResponseField>
    
    <ResponseField name="contradictory_information" type="array" required>
      Array of conflicting reports or disputed information
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="raw_results" type="array" required>
  Array of underlying search results used for the analysis
  
  <Expandable title="Raw Result Object">
    <ResponseField name="title" type="string" required>
      The title of the source article or page
    </ResponseField>
    
    <ResponseField name="snippet" type="string" required>
      Brief description or excerpt
    </ResponseField>
    
    <ResponseField name="url" type="string" required>
      The URL of the source
    </ResponseField>
    
    <ResponseField name="relevance_score" type="number">
      Relevance rating on a 1-10 scale (higher is more relevant)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="summary" type="string" required>
  Executive summary of the entire analysis
</ResponseField>

<ResponseField name="total_results" type="number" required>
  The total number of raw results analyzed
</ResponseField>

<ResponseField name="search_time" type="string" required>
  ISO8601 timestamp of when the search was completed
</ResponseField>

<ResponseField name="source" type="string" required>
  The source identifier for this response. Always `"grok-comprehensive-analysis"` for successful comprehensive mode searches.
</ResponseField>

<ResponseField name="citations" type="array" required>
  Array of citation URLs referenced throughout the analysis
</ResponseField>

<ResponseField name="citation_metadata" type="array" required>
  Array of citation metadata objects. See [Basic Response Format](/api/basic-response#response-structure) for citation metadata structure.
</ResponseField>

  "related_context": "Analysis could not be properly extracted from the response",
  "multiple_perspectives": [],
  "implications": {},

    "unconfirmed_claims": [],
    "contradictory_information": []
  },

## Notes

- Comprehensive analysis responses are cached for 30 minutes to improve performance
- All dates and timestamps use ISO8601 format
- The `comprehensive_analysis` field typically contains 500+ words of detailed analysis
- Empty arrays are returned for structured fields when analysis extraction fails
- The `source` field distinguishes between successful analyses (`"grok-comprehensive-analysis"`) and fallback responses (`"grok-comprehensive-analysis-fallback"`)
